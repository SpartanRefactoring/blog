The design of programming languages is different depending on the paradigm they
follow. However some elements are in common. 
Programming languages usually present the following schema.
There are different roles (checking the syntax,etc.)

\begin{itemize}
 \item Syntax
 \item Semantics
\end{itemize}

\begin{itemize}
 \item Cookbook (?)
 \item Protolog (?)
 \item LOLA
\end{itemize}

\subsection{LOLA}
\label{subsec:LOLA}

LOLA (Language of Language Augmentation)

Element of programming languages are ...

\begin{itemize}
 \item Commands (see Yoss's book)
 \item See specifically how smalltalk deals with commands
\end{itemize}


There are three kind of commands in \Smalltalk: unary, binary and keyword
as reported in Fig. \ref{fig:Smalltalk's-messages}. We can obtain the same 
behaviour using a fluent API in Java.

\begin{figure}
\label{fig:Smalltalk-messages}
\caption{Smalltalk's messages}
  \begin{java}{Smalltalk}
    "Unary message"
    4 negated  "a message-send in Smalltalk"
    
    "Binary message"
    5 + 3
    
    "Keyword message"
    'elephant' copyFrom: 3 to: 5 
  \end{java}
\end{figure}
